Getting Started

So you'd like to help contribute to the Grey H@t Root the Box?! That's great! We'd love to have your help! We've even made efforts to make contributing to the project as easy as possible! At this point in time, if you'd like to contribute, you must create an entire 'box', or group of flags, at a time. This box can be specified using a simple XML schema.

In order to contribute, take a look at the XML data objects described below and the sample XML file. From these, you should be able to produce an XML file that is valid for our system. Once you have created a valid XML file and have the necessary image files to go along with it, get in touch with any of the individuals listed in the who to contact section of this page.

Please pay special attention to the information listed under

XML Data Objects

XML files that describe boxes in the Grey H@t RTB contain all of the following objects. All of these values should be supplied according to the specification below. An example of a file containing all of the necessary information for uploading into the Grey H@t RTB can be found in the sample XML file section.

Box

Conceptually, the box is the container for a series of challenges. This is the root object that will be contributed for a single XML file.

  • level - The level that the box should be added to. Valid values are 0 - 9.
  • name - The name to give the box.
  • difficulty - A string representing the difficulty of the level (e.g. Easy, Medium Hard).
  • avatar - The name of an image file (jpeg, png) to associate with the box. Aspect ratio should be 2w*1h. This image file must be supplied alongside the XML document.
  • description - A description of the box and the flags contained within it. This is the meat of the challenges, and should contain enough instruction for users to obtain the contained flags without requiring additional instruction.
  • sponsor - If the box is sponsored, this will contain information regarding the box's sponsor. Sponsors can be individuals or organizations.
  • corporation - The corporation that the box should be affiliated with. Logically, this is just a grouping of boxes (for instance, all crypto-based boxes should have the corporation named 'Introduction to Crypto' listed here).
  • flags - A list of flag objects to associate with the box.
  • resources - A list of resource objects to associated with the box. These resources should be links to pages containing information regarding the box's challenges.
Sponsor

The Grey H@t RTB implementation allows for individuals and organizations to sponsor the box. Sponsors will receive attribution for their work on the box's page as well as the sponsors page.

  • name - The name of the sponsor.
  • description - Description of the sponsor.
  • url - A URL where users can get more information about the sponsor.
  • logo - The name of an image file (jpeg, png) to associate with the sponsor.
Corporation

Corporations allow for a logical grouping of boxes. For instance, all crypto-based boxes have the corporation 'Cryptographers Anonymous'. This allows for similar challenges to be related across different game levels.

  • name - The name of the corporation to associate with the box.
  • description - A description of the corporation to associate with the box.
Flag

A box can have up to ten flags. How to obtain each of these flags should be described thoroughly in the box's description tag. Capturing a flag is done by supplying a character string through an HTML form on the box page. This string must match the string contained in the 'token' value below.

  • name - A name to associate with the flag.
  • token - The token that must be submitted to capture the flag.
  • description - A short description of the flag.
  • value - A value ranging from 0.0-1.0 the reflects the portion of box points to associate with this flag. The cumulative values across all flags associated with a single box must sum up to 1.0.
  • case - A boolean depicting whether or not the value supplied in 'token' should be considered case-sensitive.
Resource

We encourage the educational aspect of this project, and as such like to have links to additional resources that have educational material relating to the contents of the current box.

  • url - The URL for the resource.
  • tag - A link text to place on the box page.
  • description - A detailed description regarding the contents of the page the link points to.

As per Wikipedia, cryptography is "the practice and study of techniques for secure communication in the presence of third parties." On a basic level, it is the idea of encoding data in such a way that (hopefully) ensures that only allowed parties can view the data. It is the digital version of the physical safe!

The foundations of modern cryptography are many times more complex than historical attempts at cryptography - modern crypto uses number theory and mathematical proofs to guarantee certain levels of privacy. However, crypto has been used as far back as 1,900 B.C. to help secure information from prying eyes!

The flags on this box cover some of the most elementary and well-known ciphers out there. First, let's take a look at the Caesar (or shift) cipher.

--- Caesar Cipher ---

Chances are that you have used a Caesar cipher at some point in your life. If you've ever seen the movie 'A Christmas Story,' you've seen one in action! A Caesar cipher is where the alphabet in which a message is written is shifted in a direction by a number of spaces. For instance, a Caesar cipher of +5 would take each letter of the alphabet and replace it with the letter that is 5 spots to the right of it. This would result in the following transformations:

A -> F
B -> G
C -> H
D -> I
...
Z -> E

Using this Caesar cipher of +5, let's encrypt the phrase 'helloworld' with it:

'helloworld' - original
'mjqqtbtwqi' - encrypted with Caesar cipher of +5

It's as simple as that! Needless to say, a Caesar cipher does not provide very much security!

Additional Schema Information

As this project is still a work-in-progress, there are a few things that we note here that would not fit well elsewhere.

  • The name of the box must be unique to the game! If it is not unique, it will not be uploaded.
  • Images supplied for the box image and sponsor image must be png or jpeg, and must be of roughly the aspect ratio 2:1.
  • Supplying a corporation name that already exists will result in the pre-existing corporation being associated with the box.
  • Supplying a sponsor name that already exists will result in the pre-existing sponsor being associated with the box.
